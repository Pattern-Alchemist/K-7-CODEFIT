# Internationalization (i18n) Setup

## Overview

AstroKalki supports multiple languages using `next-intl` for seamless internationalization.

## Supported Languages

- **English** (en) - Default
- **Spanish** (es)
- **Hindi** (hi)
- **Portuguese** (pt)

## Adding New Translations

1. Create a new JSON file in `i18n/messages/[locale].json`
2. Add the locale to `localeConfig.locales` in `i18n/config.ts`
3. Translations are automatically loaded for each locale

## Using Translations in Components

### Server Components
\`\`\`tsx
import { getTranslations } from "next-intl/server"

export default async function Page() {
  const t = await getTranslations()
  
  return <h1>{t("hero.title")}</h1>
}
\`\`\`

### Client Components
\`\`\`tsx
"use client"

import { useTranslations } from "next-intl"

export default function Component() {
  const t = useTranslations()
  
  return <h1>{t("hero.title")}</h1>
}
\`\`\`

## Language Switcher

Use the `LanguageSwitcher` component to let users change languages:

\`\`\`tsx
import { LanguageSwitcher } from "@/components/language-switcher"

export default function Header() {
  return (
    <header>
      <LanguageSwitcher />
    </header>
  )
}
\`\`\`

## URL Structure

- `/en` or `/` - English (default)
- `/es` - Spanish
- `/hi` - Hindi
- `/pt` - Portuguese

## Translation Keys

Translation keys follow a hierarchical structure:

\`\`\`
{
  "common": {...},
  "hero": {...},
  "pricing": {...},
  "booking": {...},
  "footer": {...}
}
\`\`\`

## Best Practices

1. **Consistent Keys**: Use lowercase with dots for nesting
2. **Context**: Provide enough context in keys for translators
3. **Placeholders**: Use interpolation for dynamic content:
   \`\`\`tsx
   t("booking.success", { name: "John" })
   \`\`\`
4. **Dates/Numbers**: Use `next-intl` for locale-specific formatting
5. **Missing Translations**: Always provide a fallback in the default language

## Extracting Translations

To extract all translation keys:

\`\`\`bash
pnpm i18n:extract
\`\`\`

## Translation Workflow

1. Developer creates new feature with English text
2. Keys are extracted to translation management tool
3. Professional translators provide translations
4. Translations are merged back into JSON files
5. QA tests all languages
\`\`\`
